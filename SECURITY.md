# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly.

**Do NOT open a public issue for security vulnerabilities.**

Instead, please email security concerns to the repository maintainer.

## What We Consider a Security Issue

- Sensitive information exposed in documentation (real credentials, IP addresses, API keys)
- Malicious links or content in tool documentation
- Anything that could put users at risk

## What We Do NOT Consider Security Issues

- Documentation about offensive security tools (this is an educational resource)
- Tool usage examples that show attack techniques (this is the purpose of the repo)
- Requests to remove specific tool documentation

## Response Time

We aim to respond to security reports within 48 hours.

## Disclosure Policy

We follow responsible disclosure. We will work with you to understand and address the issue before any public disclosure.
